## 第一优先级：进度追踪系统

### 1. 训练记录持久化

**目标**：每次训练的重量/次数/组数/时长存入数据库

**技术方案**：
- [x] 设计数据库表结构（workout_logs 已存在）
- [x] 创建训练记录 API（POST /api/workout）
- [x] 创建获取训练记录 API（GET /api/workout）
- [x] 前端训练完成时调用保存 API
- [x] 记录字段完整保存

### 2. 数据可视化仪表板

**目标**：Dashboard 中新增统计卡片和图表

**技术方案**：
- [x] 选择图表库（Recharts 已安装）
- [x] 创建统计数据 API（GET /api/workout/stats）
- [x] Dashboard 新增真实数据展示：
  - 总训练次数
  - 总训练时长（分钟）
  - 连续训练天数
  - 最长连续天数
  - 本周训练次数
  - 总组数

### 3. 训练完成率统计

**目标**：展示用户的训练完成情况

**技术方案**：
- [ ] 统计 API 增加完成率字段
- [ ] 周目标设定功能（可选）
- [ ] 进度条/环形图展示

### 4. 体重/体脂趋势图

**目标**：用户可手动记录身体数据

**技术方案**：
- [ ] 设计数据库表（body_metrics）
- [ ] 身体数据录入 API
- [ ] 身体数据查询 API
- [ ] 趋势折线图展示
- [ ] Dashboard 新增身体数据卡片
