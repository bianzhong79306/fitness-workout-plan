# FitPlan Pro - 健身训练计划平台需求文档

## 1. 项目概述

### 1.1 项目名称
**FitPlan Pro** - 智能健身训练计划平台

### 1.2 项目愿景
打造一个集个性化训练计划生成、进度追踪、社区互动于一体的健身平台，帮助用户科学健身，实现健康目标。

### 1.3 核心价值主张
- **智能化**：AI驱动的个性化训练计划生成
- **专业化**：由认证教练审核的训练方案
- **社交化**：健身社区激励用户坚持
- **数据化**：可视化进度追踪与分析

## 2. 目标用户

### 2.1 主要用户群体

| 用户类型 | 描述 | 核心需求 |
|---------|------|---------|
| 健身新手 | 零基础入门用户 | 简单易懂的指导、基础动作教学 |
| 增肌人群 | 追求肌肉增长 | 力量训练计划、营养建议 |
| 减脂人群 | 追求体重管理 | 有氧+力量组合、卡路里追踪 |
| 运动爱好者 | 有一定健身基础 | 进阶训练、多样化课程 |
| 健身教练 | 专业健身从业者 | 客户管理、课程发布 |

### 2.2 用户画像示例

**画像1：健身新手小明**
- 年龄：25-35岁
- 痛点：不知道如何开始，缺乏动力
- 需求：简单入门计划、视频教程、提醒功能

**画像2：健身爱好者李强**
- 年龄：20-30岁
- 痛点：训练遇到瓶颈，缺乏专业指导
- 需求：进阶计划、数据分析、专业咨询

## 3. 核心功能模块

### 3.1 用户系统
- [ ] 用户注册/登录（邮箱、手机、社交账号）
- [ ] 个人资料管理
- [ ] 身体数据记录（身高、体重、体脂率等）
- [ ] 健身目标设定（增肌/减脂/塑形/康复）
- [ ] 健身水平评估问卷

### 3.2 训练计划系统（核心）
- [ ] 智能计划生成器
  - 基于用户目标、水平、时间生成个性化计划
  - 支持周期化训练编排
  - 自动调整训练强度
- [ ] 预设计划库
  - 按目标分类（增肌/减脂/力量/耐力）
  - 按器械分类（徒手/哑铃/杠铃/健身房）
  - 按时间分类（15分钟/30分钟/60分钟）
- [ ] 自定义计划创建
  - 拖拽式训练编排
  - 动作库搜索与选择
  - 计划分享功能

### 3.3 动作库系统
- [ ] 500+专业动作视频库
- [ ] 动作分类（胸/背/肩/手臂/核心/腿/有氧）
- [ ] 动作详情（目标肌群、难度、注意事项）
- [ ] 动作搜索与筛选
- [ ] 动作替代推荐

### 3.4 训练执行模块
- [ ] 训练日历视图
- [ ] 训练计时器（组间休息/总时长）
- [ ] 训练记录（重量/次数/组数）
- [ ] 实时动作指导
- [ ] 训练完成打卡

### 3.5 进度追踪系统
- [ ] 数据可视化仪表板
- [ ] 体重/体脂趋势图
- [ ] 训练完成率统计
- [ ] 力量进步曲线
- [ ] 身体围度记录
- [ ] 前后对比照片上传

### 3.6 社区功能
- [ ] 动态发布/点赞/评论
- [ ] 健身打卡分享
- [ ] 挑战赛活动
- [ ] 排行榜系统
- [ ] 关注/粉丝系统

### 3.7 教练功能（后期）
- [ ] 教练认证申请
- [ ] 在线课程发布
- [ ] 1对1咨询服务
- [ ] 学员管理后台

## 5. 技术架构

### 5.1 技术栈选型

**前端**
- 框架：Next.js 14 (React)
- UI库：Tailwind CSS + shadcn/ui
- 状态管理：Zustand
- 图表：Recharts
- 视频播放：Video.js / Plyr

**后端**
- 框架：Node.js + Express / NestJS
- 数据库：PostgreSQL + Redis
- ORM：Prisma
- 认证：JWT + OAuth 2.0

**基础设施**
- 部署：Vercel (前端) + Railway/Render (后端)
- 存储：AWS S3 (视频/图片)
- CDN：Cloudflare
- 监控：Sentry + PostHog

## 6. 数据模型

### 6.1 核心数据实体

```sql
-- 用户表
User {
  id: UUID
  email: String
  password_hash: String
  name: String
  avatar: String
  membership_type: Enum (free, premium, pro)
  created_at: DateTime
  updated_at: DateTime
}

-- 用户身体数据
BodyMetrics {
  id: UUID
  user_id: FK
  weight: Decimal
  body_fat: Decimal
  height: Decimal
  chest: Decimal
  waist: Decimal
  hip: Decimal
  arm: Decimal
  thigh: Decimal
  recorded_at: DateTime
}

-- 训练计划
WorkoutPlan {
  id: UUID
  creator_id: FK (User)
  name: String
  description: Text
  goal: Enum (muscle_gain, fat_loss, strength, endurance, general)
  difficulty: Enum (beginner, intermediate, advanced)
  duration_weeks: Int
  sessions_per_week: Int
  equipment: String[]
  is_premium: Boolean
  is_published: Boolean
  created_at: DateTime
}

-- 训练课程
WorkoutSession {
  id: UUID
  plan_id: FK
  name: String
  day_of_week: Int
  duration_minutes: Int
  exercises: JSON
}

-- 动作库
Exercise {
  id: UUID
  name: String
  category: Enum (chest, back, shoulders, arms, core, legs, cardio)
  muscle_groups: String[]
  equipment: String
  difficulty: Enum
  video_url: String
  instructions: Text
  tips: Text
}

-- 训练记录
WorkoutLog {
  id: UUID
  user_id: FK
  session_id: FK
  started_at: DateTime
  completed_at: DateTime
  exercises: JSON
  notes: Text
  rating: Int
}
```

## 9. 非功能性需求

### 9.1 性能要求
- 页面加载时间 < 2秒
- API响应时间 < 200ms (P95)
- 视频首帧时间 < 1秒
- 支持10,000并发用户

### 9.2 安全要求
- HTTPS全站加密
- 敏感数据加密存储
- SQL注入/XSS防护
- 定期安全审计
- GDPR合规（用户数据导出/删除）

### 9.3 可用性要求
- 99.9% SLA
- 自动备份（每日）
- 灾难恢复（RTO < 4小时）

### 9.4 SEO优化
- 语义化HTML结构
- 服务端渲染（SSR）
- 结构化数据标记
- sitemap自动生成
